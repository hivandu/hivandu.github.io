18. 根据垂直需求微调模型

大家好，我是茶桁。

最近事情太多，这一节课更新的有些晚了。

首先我们先了解一下我们本节课讲要讲一些什么，我们之前介绍过 llama-index 和 LangChain，学习了将大语言模型和自己的知识库组合来解决问题的方法。这个方法中，我们不需要调整我们使用的模型，而是使用嵌入向量索引我们的数据，并在需要时查询索引来解决问题。

然而，我们也可以完全利用自己的数据，创建一个新的模型来解决问题。这种方法是OpenAI提供的微调模型功能。这也是我们要探讨的大语言模型的最后一个主题。

如何微调模型

我们都知道，AI其实是建立在大语言模型之上的，而模型再如何补全，也没有办法全知全能。在很多时候，AI所回答的内容常常错漏百出，甚至于一些垂直领域可能完全词不达意。这些其实都是因为缺少了特定领域的训练数据，而我们要做的，就是要补全这一部分数据进行训练，为我们自己的需求微调出一个擅长本领域的模型。

微调之后，我们就可以直接对模型进行提问，而不再像之前使用的llama-index或者LangChain那样，需要先通过Embedding来查询相关资料，然后再把查找到的资料一并提交给OpenAI来获得所需要的答案。

其实，OpenAI 模型微调的过程并不像我们想象的那么复杂。只需将数据提供给 OpenAI，整个微调过程就在云端的“黑盒子”中完成。需要提供的数据格式为一个文本文件，每一行都是一个 Prompt，以及对应这个 Prompt 的 Completion 接口生成的内容。

模型成本

还记得咱们之前的课程选择的模型吗？大部分时候咱们选择的都是Davinci, 而这一次我却选择了Curie, 原因就是Curie更便宜一点：

具体的可以在这里查看官方数据: https://openai.com/pricing#language-models

使用微调模型的成本比使用 OpenAI 内置的模型高得多。使用基于 Davinci 微调的模型，每使用 1000 个 Token 的成本为 0.12 美元，是使用内置的 text-davinci-003 的 6 倍，是我们最常用的 gpt-3.5-turbo 的 60 倍。因此，对于一般的文本应用来说，成本非常高。即使我们选择基于 Curie 微调，1000 个 Token 的成本也在 0.012 美元，虽然比 text-davinci-003 要便宜，但也是 gpt-3.5-turbo 的 6 倍。

模型微调的效果可以通过 OpenAI 的 "fine_tunes.results" 命令来查看。为此，我们需要提供微调任务的 ID，该 ID 可以在 "fine_tunes.list" 中列出的 fine_tunes 模型的 ID 参数中找到。

小结

今天的课程到此结束。我们学到了 OpenAI 大型语言模型中的最后两个功能。第一个功能是模型微调，可以根据自己专有的垂直领域的数据，生成一个专属于我们自己的模型，并在这个模型上不断进行微调和迭代。不过，微调后的模型使用成本比较高。另一个功能是流式地数据生成，可以为用户提供更好的交互体验。现在的大型语言模型几乎是“万能”的，可以用作机器学习的输入数据，也可以直接让它自己决定调用什么 API，以及如何解决用户的问题。希望这门课程对你有所帮助，感谢你的耐心学习！
