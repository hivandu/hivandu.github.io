21. 尝试制作你自己的数字人进行播报

Hi， 大家好。我是茶桁。

在之前的课程中，我们接触了AI进行文字回复，语音合成。

那么将这两个组合在一起，我们基本就可以制作一个智能的语音聊天机器人了。看过电影《Her》的同学都应该清楚，AI因为用了女神斯嘉丽.约翰逊的配音，吸引到了不少的观众。

不过， 我们怎么能就满足于此呢，从文字到音频，我们似乎还缺少了一点什么。是啊，谁不希望拥有一个特定的虚拟人来发出自己特定的语音。看着自己在镜头面前侃侃而谈的样子，是不是想想就兴奋？

把这些需求都结合在一起，那就是“数字人”了，我相信各位小伙伴或多或少都已经接触过，至少在抖音上看到过其他主播的“数字人”了。但是我们不得不说，那些都是一些商业公司的成熟方案，而咱们要实现的内容肯定比不了人家，但是作为概念演示，那是完全够用了。

D-iD的API

在这里，我要向大家推出D-iD，这家公司不知道有多少小伙伴听过并使用过呢。我之前在抖音上的一个视频就是使用它来完成的，不过我用的是界面服务。这家公司为免费用户提供了5分钟的免费额度。

除了界面服务之外，D-iD还提供了API服务，提供了20次调用API的机会。相比较OpenAPI所需要的复杂操作才能使用，D-iD就简便太多了，只要你有邮箱就可以进行注册。

平价的开源替代方案

算算我们之前所使用的方案，ChatGPT + D-iD + Azure，这些虽然都有免费的额度，但是完全不够用。真要是付费的话没有一个便宜的。

还记得咱们曾经使用ChatGLM来作为ChatGPT的替代方案吗？语音识别我们也可以使用本地的Whisper模型，语音合成呢，我们可以通过PaddleSpeech里的fastspeech2的开源模型来完成。似乎接下来，我们需要为视频来寻找开源的解决方案了。

目前来说，比较容易找到的解决方案，就是百度PaddlePaddle下的PaddleBobo开源项目。这个项目背后使用的是PaddleGAN的对抗生成网络算法，来实现唇形和表情的匹配。这里需要提醒一下，这个项目目前已经很久不更新了，对Python3.10的支持和依赖都有一些问题，我们在这里做一个简单的样式。

由于PaddleBobo项目有一段时间没有更新维护了，所以对于最新版本的PaddleSpeech可能存在一些小的兼容性问题。为了解决这个问题，你需要调整PaddleTools中的TTS.py文件，并修改import MODEL_HOME包的名称，以适配新的版本。这样做之后，你就可以顺利使用最新版本的PaddleSpeech并继续进行你的项目了。

小结

本节课到这里就结束了，让我们回顾一下所学的知识。我们整合前两讲的内容，成功打造了一个能够通过语音进行交互的聊天机器人。进一步地，我们还使用了 D-ID.com 这个 SaaS，让我们的聊天机器人可以通过有表情、真实的数字人形象来回答问题。然而，我们也了解到 D-ID.com 的价格相对较高，而且对 API 调用次数和视频生成数量都有限制。

为了解决这些问题，我们尝试了开源的 PaddleBobo 项目，通过文本生成带有真实口型的口播视频。此外，我们还将语音识别从云服务换成本地的 Whisper 模型，并将 ChatGPT 替换为之前测试过的开源 ChatGLM，从而获得了一个完全开源的数字人解决方案。值得一提的是，我们演示的数字人效果还比较一般，但请记住，我们并没有使用任何数据对模型进行微调，仅仅使用了预训练的开源模型，而且编写演示代码只花费了一两个晚上的时间。

值得庆幸的是，大部分深度学习技术现在已经进入了一个重要的拐点，任何人都可以通过云服务的 API 和开源模型构建自己的AI产品。这给创意和创新提供了充足的空间。希望通过这一讲的学习，你拥有足够的知识和创意，可以制作出与众不同的AI产品。让我们一同期待未来AI技术的不断发展和应用，创造出更加优秀和引人注目的产品！
